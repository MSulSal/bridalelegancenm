# Project Docs

This folder contains public project documentation that can be committed and shared.

## Files

- `CONTENT_UPDATES.md`: How to update site copy, spotlight content, and structured data.
- `DEPLOYMENT.md`: Deployment flow for Vercel + GoDaddy DNS.
- `CLIENT_HANDOFF.md`: What to hand to the client at launch and post-launch.
- `DESIGN_REFERENCES.md`: Inspiration sources and non-copying rules.
- `APPOINTMENT_ARCHITECTURE.md`: Booking strategy comparison and recommendation.
- `INTERACTION_PLAN.md`: Planned interactive UX features and rollout order.

## Documentation Rules

- Keep private/internal-only notes in `.private/`, not in `docs/`.
- Record architecture decisions as they happen, not at the end.
- Prefer short, practical notes over long narrative text.
